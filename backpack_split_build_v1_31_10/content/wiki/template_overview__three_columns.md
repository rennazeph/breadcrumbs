Column A is navigation and authoring control. It should remain compact, predictable, and tolerant of growth. Cascade navigation works here because it provides structure without depending on color alone: the separator and selected marker still communicate state in monochrome displays.

Column B is the primary reading surface. It carries the title, page description, and subtitle-based Markdown fragments. This prevents a long page from becoming one unbroken document, while still letting the reader scroll naturally through the selected subject.

Column C is an infobox. It contains a top image and a smaller Markdown note. That makes it useful for definitions, source notes, warnings, or small metadata without interrupting the main article.
